# 回旋镖任务（Orchestrator 模式）

回旋镖任务功能（也称为 Orchestrator 模式）允许 Roo Code 自动执行多步骤工作流程。当您启动一个回旋镖任务时，Roo 会完全自主地执行整个任务，期间不会要求您进行任何交互，直到任务完成或需要人工干预时才会返回结果。

## 使用方法

1. 切换到 Orchestrator 模式（AKA 回旋镖模式）
2. 向 Roo 描述您想要执行的任务
3. Roo 会自动执行任务并在完成后返回结果

## 工作原理

当您在 Orchestrator 模式下启动任务时，Roo 会分析任务需求并创建一个执行计划。然后 Roo 会按照计划自动执行所有步骤，包括读取文件、编辑代码、运行命令等，直到任务完成。

## 关键注意事项

- 回旋镖任务在 Orchestrator 模式下运行，该模式具有广泛的工具访问权限
- 任务执行期间不会要求用户交互，直到任务完成或遇到错误
- 您可以随时停止正在运行的回旋镖任务

## 常见问题解答

### 回旋镖任务和普通任务有什么区别？

普通任务在执行过程中可能需要用户批准每个步骤，而回旋镖任务会自动执行所有步骤，无需用户交互，直到任务完成。

### 如何停止正在运行的回旋镖任务？

您可以通过点击 Roo Code 界面中的停止按钮来停止正在运行的回旋镖任务。

### 回旋镖任务可以访问哪些工具？

回旋镖任务可以访问 Orchestrator 模式下的所有工具，包括文件读写、命令执行、网络请求等。
