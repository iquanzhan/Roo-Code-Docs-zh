# 浏览器使用

Roo Code 可以使用 Puppeteer 控制的浏览器与网站进行交互。当您要求 Roo 访问网站时，浏览器会自动启动，捕获屏幕截图，并与网页元素进行交互。

您可以通过以下配置设置来调整浏览器自动化功能：

- **视口大小**：设置浏览器窗口的宽度和高度
- **截图质量**：设置截图的压缩质量
- **远程浏览器连接**：连接到远程浏览器实例

## 浏览器如何工作

默认情况下，Roo Code 使用内置浏览器，该浏览器：
- 在您要求访问网站时自动启动
- 捕获网页截图
- 允许 Roo 与网页元素交互
- 在后台无形运行

所有这些都直接在 VS Code 内完成，无需任何设置。

## 使用浏览器功能

典型的浏览器交互遵循以下模式：

**重要提示：** 浏览器使用功能需要 Claude Sonnet 3.5 或 3.7 模型。

1. 要求 Roo 访问网站
2. Roo 启动浏览器并向您显示截图
3. 请求额外操作（点击、输入、滚动）
4. Roo 在完成后关闭浏览器

例如：

```
打开浏览器并查看我们的网站。
```

```
你能检查一下我的网站 https://roocode.com 是否正确显示吗？
```

```
浏览 http://localhost:3000，向下滚动到页面底部，检查页脚信息是否正确显示。
```

## 浏览器操作如何工作

browser_action 工具控制一个浏览器实例，在每次操作后返回截图和控制台日志，让您能够看到交互结果。

主要特点：
- 每个浏览器会话必须以 `launch` 开始，以 `close` 结束
- 每条消息只能使用一个浏览器操作
- 浏览器激活时，不能使用其他工具
- 在执行下一个操作之前，必须等待响应（截图和日志）

### 可用的浏览器操作

| 操作 | 描述 | 使用时机 |
|--------|-------------|------------|
| `launch` | 在 URL 处打开浏览器 | 启动新的浏览器会话 |
| `click` | 在特定坐标点击 | 与按钮、链接等交互 |
| `type` | 在活动元素中输入文本 | 填写表单、搜索框 |
| `scroll_down` | 向下滚动一页 | 查看折叠内容 |
| `scroll_up` | 向上滚动一页 | 返回之前的内容 |
| `close` | 关闭浏览器 | 结束浏览器会话 |

## 浏览器使用配置/设置

:::info 默认浏览器设置
- **启用浏览器工具**：已启用
- **视口大小**：小桌面 (900x600)
- **截图质量**：75%
- **使用远程浏览器连接**：已禁用
:::

### 访问设置

要更改 Roo 中的浏览器/计算机使用设置：

1. 通过点击齿轮图标 <Codicon name="gear" /> → 浏览器/计算机使用 打开设置

   <img src="/img/browser-use/browser-use.png" alt="浏览器设置菜单" width="600" />

### 启用/禁用浏览器使用

**目的**：主开关，使 Roo 能够使用 Puppeteer 控制的浏览器与网站交互。

要更改此设置：
1. 在浏览器/计算机使用设置中勾选或取消勾选"启用浏览器工具"复选框

   <img src="/img/browser-use/browser-use-2.png" alt="启用浏览器工具设置" width="300" />

### 视口大小

**目的**：确定 Roo Code 使用的浏览器会话分辨率。

**权衡**：较高的值提供更大的视口但会增加令牌使用量。

要更改此设置：
1. 在浏览器/计算机使用设置中点击"视口大小"下的下拉菜单
2. 选择可用选项之一：
   - 大桌面 (1280x800)
   - 小桌面 (900x600) - 默认
   - 平板 (768x1024)
   - 移动设备 (360x640)
2. 选择所需的分辨率。

   <img src="/img/browser-use/browser-use-3.png" alt="视口大小设置" width="600" />

### 截图质量

**目的**：控制浏览器截图的 WebP 压缩质量。

**权衡**：较高的值提供更清晰的截图但会增加令牌使用量。

要更改此设置：
1. 在浏览器/计算机使用设置中调整"截图质量"下的滑块
2. 设置 1-100% 之间的值（默认为 75%）
3. 较高的值提供更清晰的截图但会增加令牌使用量：
   - 40-50%：适用于基本的基于文本的网站
   - 60-70%：适用于大多数一般浏览
   - 80%+：在视觉细节至关重要的时候使用

   <img src="/img/browser-use/browser-use-4.png" alt="截图质量设置" width="600" />

### 远程浏览器连接

**目的**：将 Roo 连接到现有 Chrome 浏览器而不是使用内置浏览器。

**优势**：
- 在容器化环境和远程开发工作流中工作
- 在浏览器使用之间保持认证会话
- 消除重复的登录步骤
- 允许使用具有特定扩展的自定义浏览器配置文件

**要求**：Chrome 必须在启用远程调试的情况下运行。

要启用此功能：
1. 在浏览器/计算机使用设置中勾选"使用远程浏览器连接"框
2. 点击"测试连接"进行验证

   <img src="/img/browser-use/browser-use-5.png" alt="远程浏览器连接设置" width="600" />

#### 常见使用场景

- **开发容器**：从容器化的 VS Code 连接到主机 Chrome 浏览器
- **远程开发**：将本地 Chrome 与远程 VS Code 服务器一起使用
- **自定义 Chrome 配置文件**：使用具有特定扩展和设置的配置文件

#### 连接到可见的 Chrome 窗口

连接到可见的 Chrome 窗口以实时观察 Roo 的交互：

**macOS**
```bash
/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome --remote-debugging-port=9222 --user-data-dir=/tmp/chrome-debug --no-first-run
```

**Windows**
```bash
"C:\Program Files\Google\Chrome\Application\chrome.exe" --remote-debugging-port=9222 --user-data-dir=C:\chrome-debug --no-first-run
```

**Linux**
```bash
google-chrome --remote-debugging-port=9222 --user-data-dir=/tmp/chrome-debug --no-first-run
```
