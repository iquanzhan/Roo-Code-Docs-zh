# 检查点

检查点功能为您的代码提供自动版本控制，确保在进行重大更改之前始终可以恢复到以前的状态。此功能通过创建代码库的快照来工作，您可以随时查看差异或恢复到任何检查点。

## 配置选项

您可以通过以下配置选项来调整检查点功能：

- **检查点间隔**：设置自动创建检查点的时间间隔
- **最大检查点数**：设置保留的检查点数量上限
- **检查点触发条件**：设置触发检查点创建的条件

## 工作原理

当您编辑代码时，Roo Code 会自动创建检查点。这些检查点包含您的代码库在特定时间点的完整快照。您可以随时查看这些检查点之间的差异，或恢复到任何检查点。

## 使用方法

### 查看差异

您可以通过以下步骤查看检查点之间的差异：

1. 打开检查点视图
2. 选择要比较的两个检查点
3. 查看差异

### 恢复检查点

您可以通过以下步骤恢复到特定检查点：

1. 打开检查点视图
2. 选择要恢复的检查点
3. 确认恢复操作

## 限制和注意事项

- 检查点功能仅在支持的文件类型上工作
- 检查点不会保存在远程仓库中
- 检查点的创建可能会占用一定的存储空间

## 技术架构

检查点功能的技术架构包括以下组件：

- **影子 Git 仓库**：用于存储检查点数据
- **检查点服务**：负责创建和管理检查点
- **UI 组件**：提供检查点视图和操作界面
