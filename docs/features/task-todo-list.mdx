# 任务待办列表

通过集成的待办事项管理功能，让您能够井井有条并专注于您的开发目标。任务待办列表提供交互式、持久的检查清单，直接在聊天界面中跟踪您完成复杂、多步骤工作流程的进度。

<img src="/img/task-todo-list/task-todo-list-3.png" alt="任务待办列表概览，显示 Roo Code 中的交互式检查清单" width="600" />

:::tip 待办列表触发器
待办列表会为复杂任务、多步骤工作流程或使用 Architect 模式时自动创建。您也可以通过要求 Roo "使用 update_todo_list 工具" 或 "创建待办列表" 来手动触发它们。

有关自动和手动触发器的详细信息，请参见 [Roo 何时创建待办列表](#when-roo-creates-todo-lists)。
:::

## 使用案例

**之前**：在脑海中或单独的笔记文件中手动跟踪复杂任务的步骤，很容易跟丢进度和下一步。

**使用任务待办列表**：Roo Code 会自动创建并管理嵌入在对话中的结构化检查清单。您可以查看每个项目的进度，观察 AI 标记项目完成，并在 Roo 提出更新以供批准时提供反馈。

## 工作原理

任务待办列表功能由 [`update_todo_list`](/advanced-usage/available-tools/update-todo-list) 工具提供支持，该工具可在聊天界面中实现动态任务管理。

### Roo 何时创建待办列表

Roo 通过自动检测和手动请求创建待办列表：

- **检测到任务复杂性** - 在您的请求中识别出多个步骤、阶段或依赖关系
- **在 Architect 模式下工作** - 始终创建待办列表作为构建工作的主要规划工具
- **直接工具请求** - 说 "使用 update_todo_list 工具" 或 "请使用 update_todo_list"

请记住：即使手动触发，Roo 仍保持对待办列表内容和工作流程的控制。您在批准对话框中提供反馈，但 Roo 根据任务需要管理列表。

### 显示和交互

待办列表出现在多个地方：

1. **任务标题摘要**：通过 `TodoListDisplay` 组件显示的紧凑、只读显示，显示进度和下一个重要项目

   <img src="/img/task-todo-list/task-todo-list-1.png" alt="任务标题摘要，显示待办列表进度" width="600" />

2. **交互式工具块**：通过 `UpdateTodoListToolBlock` 组件在聊天中的界面，允许您：
   - 查看所有待办项目及其当前状态
   - 在 Roo 提出更新以供批准时编辑项目描述
   - 使用 "编辑" 按钮暂存更改（仅在 Roo 下次更新列表时应用）
   - 查看 Roo 管理待办工作流程的进度

   <img src="/img/task-todo-list/task-todo-list-2.png" alt="聊天中的交互式待办列表工具块" width="600" />

3. **环境详情**：待办列表作为 "REMINDERS" 表出现在 environment_details 部分，让 AI 能够持久访问当前的待办状态。注意：如果 `todoListEnabled` 设置被禁用，提醒部分将不会出现在环境详情中。

### 理解任务状态

Roo Code 根据任务进度自动管理状态进展。每个待办项目有三种状态之一：

**待处理**：显示一个空复选框，表示任务尚未开始
![待处理的待办项目，带有空复选框](../../static/img/task-todo-list/task-todo-list-4.png)

**进行中**：显示一个黄色点指示器，表示任务当前正在进行中
![进行中的待办项目，带有黄色点指示器](../../static/img/task-todo-list/task-todo-list-5.png)

**已完成**：具有绿色勾号，确认任务已完全完成
![已完成的待办项目，带有绿色勾号](../../static/img/task-todo-list/task-todo-list-6.png)

## 常见问题解答

**"我可以创建自己的待办列表吗？"**
是的，您可以通过要求 Roo "使用 update_todo_list 工具" 或 "创建待办列表" 来手动触发待办列表创建。但是，Roo 保持对 待办列表内容和工作流程的控制 - 您在批准对话框中提供反馈，但 Roo 根据任务需要管理列表。

**"我可以将待办列表用于简单任务吗？"**
Roo Code 通常只为复杂、多步骤的任务创建待办列表，因为它们能提供明确的价值。对于简单任务，列表管理的开销是不必要的。

**"为什么我不能直接控制待办列表？"**
这是一个架构设计决策，其中 Roo Code 保持对任务管理的权威。您提供指导和反馈，但 Roo 控制工作流程以确保一致的任务进展和准确的状态跟踪。

## 配置

### 禁用待办列表

您可以通过 `todoListEnabled` 设置完全禁用待办列表。禁用时：
- Roo 不会为任何任务创建待办列表
- REMINDERS 部分不会出现在环境详情中
- 现有的待办列表将保持可见但不会更新

### 自动批准待办列表更新

您可以启用自动批准待办列表更新，以减少长时间工作流程中的中断。启用后，Roo 将自动更新任务进度，而无需确认每次更改。

要配置此功能，请参见 [更新待办列表自动批准设置](/features/auto-approving-actions#update-todo-list)。
